🤝 GitHub Collaborator設定・解除 手順書

クライアントのリポジトリにsonosannを招待・作業後に解除する

目的：クライアントがsonosannをCollaborator（協力者）として招待することで、sonosannがVSCodeからクライアントのリポジトリに直接プッシュできるようになる。作業完了後は必ず解除する。

A. クライアント側の操作（招待）
1
Settings → Collaborators を開く
クライアント
  • リポジトリの上部タブ Settings をクリック
  • 左メニュー Collaborators をクリック
  • 人を追加する（Add people） ボタンを押す
▲ 「人を追加する」ボタンが表示された画面
⚠️ ファイルが入っていない空のリポジトリではボタンがアクティブにならない。必ずファイルアップロード後に行うこと。
2
sonosannを検索して招待する
クライアント
  • 検索窓に kobayashi4556 と入力
  • 候補に satsuki / kobayashi4556 が表示される
  • 候補をクリックして選択状態にする
  • リポジトリに追加 ボタンを押す
▲ kobayashi4556で検索すると候補が表示される
✅ 候補をクリックしてからボタンがアクティブになります。
3
招待完了を確認する
クライアント
  • 上部バナーに 「kobayashi4556 has been added as a collaborator」 と表示される
  • Collaborators画面に 「招待待ち」 と表示される
▲ 招待完了のバナー表示
▲ sonosannの返信待ち状態
B. sonosann側の操作（Accept）
4
招待メールを確認してAcceptする
sonosann
  • GitHubから招待メールが届く
  • メール内の Accept invitation をクリック
  • ブラウザが開いてリポジトリにアクセスできる状態になる
▲「You now have push access to the repository」と表示される
✅ この状態でVSCodeからクライアントのリポジトリにプッシュできます。
5
クライアント側でも協力者になったか確認する
クライアント
  • Settings → Collaborators を開く
  • ページを再読み込み（F5）する
  • 「招待待ち」→「協力者」 に変わっていればOK
▲「kobayashi4556・協力者」と表示されればOK
C. 作業完了後の解除
6
Collaboratorを解除する
クライアント
  • Settings → Collaborators を開く
  • kobayashi4556の右端の ゴミ箱アイコン をクリック
  • 確認ダイアログの 「このリポジトリからkobayashi4556を削除する」 をクリック
▲ 削除確認ダイアログ
▲「まだ共同作業者を招待していません」に戻れば完了
✅ 解除後、sonosannはこのリポジトリにプッシュできなくなります。

🎉 Collaborator設定・解除 完了！

招待 → Accept → 作業 → 解除
この流れを毎回の案件で繰り返します。
